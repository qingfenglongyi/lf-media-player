# 数据库设计文档

## 概述
本应用使用 Room 数据库存储播放列表和歌曲信息。数据库名称为 `lf_media_player_db`。

---

## 表结构

### 1. playlists（歌单表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (主键，自增) | 歌单唯一标识 |
| name | String | 歌单名称 |
| createdAt | Long | 创建时间（时间戳） |
| updatedAt | Long | 更新时间（时间戳） |

**用途**：存储用户创建的歌单信息

**约束**：
- 主键自增
- name 字段使用 `REPLACE` 策略，相同名称会覆盖

---

### 2. playlist_songs（歌单歌曲关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| playlistId | Long（联合主键） | 关联的歌单ID |
| songId | Long（联合主键） | 歌曲ID |
| position | Int | 歌曲在歌单中的排序位置 |
| addedAt | Long | 添加到歌单的时间（时间戳） |

**用途**：存储歌单与歌曲的多对多关联关系

**约束**：
- 联合主键：`playlistId` + `songId`
- 使用 `REPLACE` 策略

---

### 3. songs（歌曲表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long（主键） | 歌曲ID（来自 MediaStore） |
| title | String | 歌曲标题 |
| artist | String | 艺术家/歌手 |
| album | String | 专辑名称 |
| duration | Long | 歌曲时长（毫秒） |
| path | String | 文件路径 |
| lastPlayedAt | Long? | 上次播放时间（时间戳，可为空） |
| isHidden | Boolean | 是否隐藏（默认 false） |

**用途**：存储歌曲的元数据信息

---

### 4. config（配置表）【新增】

| 字段 | 类型 | 说明 |
|------|------|------|
| key | String（主键） | 配置项键名 |
| value | String | 配置项值 |
| updatedAt | Long | 更新时间（时间戳） |

**用途**：存储用户配置和播放状态

**约束**：
- 主键为 String 类型的 key
- 使用 `REPLACE` 策略

**已定义的配置项**：

| key | value 格式 | 说明 |
|-----|-----------|------|
| current_playlist | JSON数组，如 `[{"songId":123,"position":0},...]` | 当前播放列表的歌曲ID和位置 |
| current_index | 整数 | 当前播放的歌曲索引 |
| play_mode | 字符串 (LIST_LOOP/SINGLE_LOOP/SHUFFLE) | 播放模式 |
| last_position | Long | 上次播放位置（毫秒） |
| last_volume | Float | 上次音量 (0.0-1.0) |
| ... | ... | 其他配置可扩展 |

**可扩展性**：
- 可以添加任意配置项，如：
  - `theme` - 主题设置
  - `equalizer_settings` - 均衡器设置
  - `playback_order` - 播放顺序偏好
  - `ui_preferences` - UI偏好设置

**JSON格式说明**：
- `current_playlist` 使用 JSON 数组存储，格式：`[{"songId":123,"position":0},{"songId":456,"position":1},...]`

## 表关系图

```
┌─────────────────┐       ┌─────────────────────┐       ┌─────────────────┐
│   playlists    │       │   playlist_songs    │       │     songs       │
├─────────────────┤       ├─────────────────────┤       ├─────────────────┤
│ id (PK)        │──┐    │ playlistId (PK,FK)  │──┐    │ id (PK)         │
│ name           │  │    │ songId (PK,FK)      │──┼──┐ │ title           │
│ createdAt      │  └───│ position            │  │  │ artist          │
│ updatedAt      │       │ addedAt            │  │  │ album           │
└─────────────────┘       └─────────────────────┘  │  │ duration        │
                                                   │  │ path            │
                                                   │  │ lastPlayedAt    │
                                                   │  │ isHidden        │
                                                   └──┴─────────────────┘

┌─────────────────┐
│     config      │
├─────────────────┤
│ key (PK)        │
│ value           │
│ updatedAt       │
└─────────────────┘
```

**关系说明**：
- `playlists` 与 `playlist_songs`：一对多（1:N）
- `playlist_songs` 与 `songs`：多对一（N:1）
- `config`：独立配置表，存储键值对

## DAO 接口设计

### PlaylistDao

| 方法 | 说明 |
|------|------|
| `getAllPlaylists(): Flow<List<Playlist>>` | 获取所有歌单（按更新时间降序） |
| `getPlaylistById(id: Long): Playlist?` | 根据ID获取歌单 |
| `getPlaylistByName(name: String): Playlist?` | 根据名称获取歌单 |
| `insertPlaylist(playlist: Playlist): Long` | 插入歌单（返回新ID） |
| `updatePlaylist(playlist: Playlist)` | 更新歌单 |
| `deletePlaylist(playlist: Playlist)` | 删除歌单 |
| `deletePlaylistById(id: Long)` | 根据ID删除歌单 |
| `getPlaylistSongs(playlistId: Long): List<PlaylistSong>` | 获取歌单中的所有歌曲（按position排序） |
| `getPlaylistSongsFlow(playlistId: Long): Flow<List<PlaylistSong>>` | 歌单歌曲流（响应式） |
| `insertPlaylistSong(playlistSong: PlaylistSong)` | 添加歌曲到歌单 |
| `deletePlaylistSong(playlistId: Long, songId: Long)` | 从歌单中删除歌曲 |
| `clearPlaylist(playlistId: Long)` | 清空歌单中的所有歌曲 |
| `getPlaylistSongCount(playlistId: Long): Int` | 获取歌单中的歌曲数量 |

### SongDao

| 方法 | 说明 |

## 注意事项

1. **编码问题**：歌词文件可能使用 GBK 或 BIG5 编码，需要尝试多种编码进行解析
2. **配置持久化**：应用退出时自动保存播放状态到 config 表，应用启动时恢复
3. **JSON格式**：播放列表使用 JSON 存储，解析时需注意异常处理
4. **hidden 字段**：仅用于隐藏通话录音等不适合显示的文件，不影响 MediaStore 的原始数据
5. **可扩展性**：config 表采用键值对设计，便于后续添加新的配置项
