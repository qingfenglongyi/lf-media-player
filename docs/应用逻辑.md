# 应用逻辑说明

## 重要规则：使用歌曲ID而非索引
- **所有与歌曲相关的参数传递必须使用歌曲ID，禁止使用歌曲索引**
- 原因：歌曲库扫描结果可能变化，使用索引会导致引用错误
- 保存播放列表时保存歌曲ID列表，恢复时通过ID匹配
- config表中的 `current_index` 实际存储的是当前播放歌曲的ID，而非列表索引位置

## 一、应用运行步骤

### 1.1 启动
1. 应用启动，显示主界面
2. 未设置音乐目录时，不扫描歌曲，显示空界面
3. 等待用户设置音乐目录

### 1.2 设置音乐目录后
1. 用户通过目录选择器选择音乐文件夹
2. 使用 SAF (Storage Access Framework) 扫描该目录
3. 将扫描到的歌曲加载到**歌曲库**
4. 同时从数据库恢复播放列表（如有保存）
5. **如有保存的播放列表则自动续播**，否则等待用户操作

### 1.3 后续启动
1. 加载已保存的音乐目录设置
2. 扫描目录加载歌曲到**歌曲库**
3. 从数据库恢复播放列表状态
4. **如有保存的播放列表则自动续播**，否则等待用户操作

### 1.4 自动续播条件
- 只有播放列表里有歌曲时才自动播放
- 从数据库恢复上次的播放位置，继续播放
- 记录播放进度以便下次恢复

## 二、歌曲来源

### 2.1 扫描方式
- 使用 SAF (Storage Access Framework) 访问用户选择的目录
- 支持格式：mp3, flac, m4a, wav, ogg, aac
- 使用 MediaMetadataRetriever 获取音频元数据（艺术家、专辑、时长）

### 2.2 歌曲存储
- **元数据**保存在应用数据库中（Room）
- **文件**本身存储在用户设备上，应用只保存文件引用

## 三、数据管理

### 3.1 歌曲库
- **定义**：用户通过 SAF 目录扫描到的所有歌曲
- **数据来源**：每次扫描目录时重新获取
- **用途**：
  - 显示所有可用歌曲
  - 支持按艺术家、专辑分类浏览
  - 支持搜索过滤
- **追加**：用户选择新目录时会重新扫描追加新歌曲（根据目录文件名判断是否重复）

### 3.2 歌单
- **定义**：用户创建的自定义歌曲集合
- **存储**：保存在 Room 数据库的 `playlists` 和 `playlist_songs` 表
- **用途**：
  - 用户管理自己喜好的歌曲分类
  - 可添加、删除、重命名歌单
  - 可在歌单中增减歌曲
- **持久化**：歌单数据会保存到数据库，卸载重装后仍保留

### 3.3 播放列表
- **定义**：当前正在播放或即将播放的歌曲队列
- **数据来源**：
  - 用户从歌曲库点击歌曲时设置
  - 用户添加到队列的歌曲
- **用途**：控制当前播放顺序
- **持久化**：保存到数据库 `config` 表，包含：
  - 播放列表内容（歌曲ID列表）
  - 当前播放歌曲ID
  - 播放模式（列表循环/单曲循环/随机）
  - 上次播放位置
  - 上次音量

## 四、数据流向图

```
[SAF目录扫描] --> [歌曲库] --> 用户点击 --> [播放列表] --> 播放器播放
                                        |
用户创建 --> [歌单] --------------------+

[启动时] <-- 数据库恢复 --> [播放列表续播]
[退出时] --> 数据库保存 --> [播放列表状态]
```

## 五、关键行为说明

### 5.1 点击播放
1. 用户点击歌曲库中的歌曲
2. 将歌曲设置为播放列表第一首
3. 播放器开始播放

### 5.2 添加到队列
1. 用户选择多首歌曲
2. 点击"添加到播放列表"
3. 所选歌曲追加到当前播放列表末尾

### 5.3 添加到歌单
1. 用户选择歌曲
2. 选择目标歌单
3. 歌曲添加到歌单（数据库持久化）

### 5.4 播放列表 vs 歌单
| 特性 | 播放列表 | 歌单 |
|------|---------|------|
| 用途 | 当前播放 | 收藏管理 |
| 生命周期 | 运行时+持久化 | 持久化 |
| 保存位置 | 内存+数据库config表 | 数据库playlists表 |
| 卸载后保留 | 是（通过config表） | 是 |

## 六、退出与恢复

### 6.1 播放列表保存机制
- 播放列表变化时（添加/删除/重排）触发定时保存
- 使用定时器，延迟30秒后保存，避免频繁保存
- 保存内容：播放列表（歌曲ID列表）、当前播放歌曲ID
- 应用退出时立即保存所有状态

### 6.2 应用退出时
1. 保存当前播放列表到数据库 `config.current_playlist`（歌曲ID列表）
2. 保存当前播放歌曲ID到数据库 `config.current_id`
3. 保存播放位置到数据库 `config.last_position`
4. 保存播放模式和音量

### 6.3 应用启动时
1. 加载歌曲库
2. 从数据库恢复播放列表（使用歌曲ID匹配）
3. 根据歌曲ID找到当前播放歌曲
4. 恢复播放位置
5. **如有保存的播放列表则自动从上次位置续播**

## 七、数据库存储说明

### 7.1 config 表配置项
| key | 说明 |
|-----|------|
| current_playlist | JSON数组，保存播放列表歌曲ID和位置 |
| current_id | 保存当前播放的歌曲ID |
| play_mode | 播放模式 (LIST_LOOP/SINGLE_LOOP/SHUFFLE) |
| last_position | 上次播放位置（毫秒） |
| last_volume | 上次音量 (0.0-1.0) |

### 7.2 数据恢复流程
1. 读取 `current_playlist` 获取歌曲ID列表
2. 根据歌曲ID从歌曲库中匹配实际歌曲
3. 根据 `current_id` 找到当前播放歌曲
4. 读取 `last_position` 跳转到上次播放位置
5. 自动播放

## 八、卸载重装后行为

1. 音乐目录设置（SharedPreferences）：**可能丢失**（取决于系统备份设置）
2. 歌单数据（Room数据库）：**可能保留**（取决于系统备份设置）
3. 播放列表状态：**可能保留**（通过config表）
4. 歌曲文件：**保留**（存储在设备上）
